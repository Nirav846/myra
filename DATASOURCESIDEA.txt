I’m turning the IAS into a two-stage system now: first detect institutional setup, then wait for the exact trigger candle so the entry is not random.


Here is a clean IAS + Entry Timing Engine design you can wire straight into MYRA.

The key idea is simple: IAS tells you what to watch; the entry engine tells you when to buy.
Do not buy on score alone.

IAS + Entry Timing Engine
1) Setup filter

A stock is tradable only when the broader structure is strong.

setup_ok = (
    IAS >= 7.0 and
    SAST_score >= 7.0 and
    Delivery_score >= 7.0 and
    Price_structure_score >= 6.0
)

Optional harder filter for high conviction:

high_conviction = (
    IAS >= 8.0 and
    (SAST_score >= 8.0 or Delivery_score >= 8.0) and
    Volatility_compression_score >= 6.0
)
2) Entry engine

Use three entry types.
This is better than forcing one universal buy point.

A. Breakout entry

Best when stock is coming out of a tight base.

Exact buy point

Buy only when all of these are true:

breakout_entry = (
    close > base_high * 1.003 and
    close > 20_day_high and
    volume >= 1.5 * avg_volume_20d and
    delivery_pct >= 45 and
    ATR_5 < ATR_20
)
Interpretation
base_high: highest close or high of the base
1.003: 0.3% buffer above breakout level
volume spike confirms demand
ATR contraction confirms stored energy
Entry price
entry_price = max(base_high * 1.003, breakout_candle_high)
Stop-loss
stop_loss = min(base_low, breakout_candle_low) * 0.995
B. Pullback entry

Best after the stock already broke out and is retesting support.

Exact buy point
pullback_entry = (
    close > 20_EMA and
    low <= 20_EMA * 1.01 and
    close >= open and
    volume <= avg_volume_10d * 0.9 and
    IAS >= 7.5
)
Interpretation
stock pulls back gently
volume dries up
no serious selling pressure
trend remains intact
Entry price

Buy on:

entry_price = close if close > intraday_reclaim_level else intraday_reclaim_level

A good reclaim level is:

the day’s high after the pullback candle
or the 5-minute high if you are executing intraday
Stop-loss
stop_loss = pullback_swing_low * 0.995
C. Bear trap / failed breakdown entry

Best when price briefly breaks support and snaps back.

Exact buy point
trap_entry = (
    low < support_level and
    close > support_level and
    delivery_pct >= 50 and
    volume >= 1.3 * avg_volume_20d and
    wick_lower > body_size
)
Interpretation
market tries to shake out weak hands
price reclaims support
high delivery shows absorption
Entry price
entry_price = close if close > trap_candle_high else trap_candle_high
Stop-loss
stop_loss = trap_candle_low * 0.995
3) Timing logic

Do not enter immediately when setup is true.
Enter only when setup + trigger + confirmation align.

should_buy = (
    setup_ok and
    (
        breakout_entry or
        pullback_entry or
        trap_entry
    )
)

To reduce false signals, add a final confirmation:

confirmation = (
    close >= VWAP and
    market_regime in ["bullish", "neutral"] and
    relative_strength_vs_index > 0
)

Final trigger:

should_buy = setup_ok and confirmation and (
    breakout_entry or pullback_entry or trap_entry
)
4) Entry ranking

If multiple stocks trigger on the same day, rank them by a composite entry quality score.

entry_quality = (
    0.35 * IAS +
    0.20 * breakout_strength +
    0.15 * delivery_quality +
    0.15 * compression_quality +
    0.15 * relative_strength
)

Where:

breakout_strength = distance above pivot with volume support
delivery_quality = delivery above 45–60%
compression_quality = tighter base, lower ATR
relative_strength = outperforming NIFTY / sector

Buy only the top-ranked names.

5) Position sizing

Size the trade from the stop-loss, not from emotion.

risk_per_trade = portfolio_value * 0.01
qty = risk_per_trade / abs(entry_price - stop_loss)

If you want tighter control:

qty = min(qty, max_position_size)
6) Full engine flow
def entry_timing_engine(stock):
    if stock.IAS < 7.0:
        return None

    setup_ok = (
        stock.SAST_score >= 7.0 and
        stock.Delivery_score >= 7.0 and
        stock.Price_structure_score >= 6.0
    )

    if not setup_ok:
        return None

    breakout = (
        stock.close > stock.base_high * 1.003 and
        stock.close > stock.high_20d and
        stock.volume >= 1.5 * stock.avg_volume_20d and
        stock.delivery_pct >= 45 and
        stock.atr_5 < stock.atr_20
    )

    pullback = (
        stock.close > stock.ema_20 and
        stock.low <= stock.ema_20 * 1.01 and
        stock.close >= stock.open and
        stock.volume <= stock.avg_volume_10d * 0.9
    )

    trap = (
        stock.low < stock.support_level and
        stock.close > stock.support_level and
        stock.delivery_pct >= 50 and
        stock.volume >= 1.3 * stock.avg_volume_20d
    )

    confirmation = (
        stock.close >= stock.vwap and
        stock.relative_strength > 0
    )

    if not confirmation:
        return None

    if breakout:
        entry = max(stock.base_high * 1.003, stock.high)
        stop = min(stock.base_low, stock.low) * 0.995
        return {"type": "breakout", "entry": entry, "stop": stop}

    if pullback:
        entry = max(stock.high, stock.ema_20)
        stop = stock.pullback_low * 0.995
        return {"type": "pullback", "entry": entry, "stop": stop}

    if trap:
        entry = max(stock.close, stock.trap_high)
        stop = stock.trap_low * 0.995
        return {"type": "bear_trap", "entry": entry, "stop": stop}

    return None
7) Practical trading rules

These make the engine much safer:

Do not buy if the stock is extended more than 5–7% above pivot.
Do not buy if the breakout candle closes weak near its low.
Do not buy if market regime is sharply bearish.
Do not buy if delivery is strong but price is under the 50-day trend.
Do not average down unless the structure is still intact.
8) Best version for MYRA

Use this sequence:

IAS filter
Setup classification
Entry trigger
Risk-based sizing
Stop-loss
Exit on failure or partial profit booking

That gives you a proper system:
smart money detection → exact buy point → disciplined risk control
