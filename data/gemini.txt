# GEMINI Instructions - MYRA Intelligence Workspace

## Core Mandate
This workspace is the home of **MYRA (Myra Yield & Research Analytics)**, an elite, local-first institutional stock screener. All logic MUST adhere to the **Modular Architecture v2.5**.

## 1. Architecture Principles (v2.5)
The system is built on a strictly isolated Seven-Layer Orchestration:
- `myra_app/screener.py`: Central coordinator and strategy pipeline execution.
- `myra_app/librarian.py`: Data lake manager (Exclusive gatekeeper for DuckDB).
- `myra_app/engine.py`: Universal Turbo-SQL engine and quantitative math layer.
- `myra_app/fetcher.py`: Smart Proxy & Multi-Source acquisition (auto-rotation of User-Agents, 24h fallback cache).
- `myra_app/index_engine.py`: Native NSE Index & VIX Engine (1-hour cached direct quotes).
- `myra_app/fundamental_manager.py`: Multi-factor data orchestration (Growth, Quality, Valuation, Risk).
- `myra_app/myra.py`: Pure UI & Navigator.

## 2. Data Flow & State Verification (Strict Validation)
Before generating or modifying any pipeline code, ALWAYS verify the complete data lifecycle:
1. **Fetch:** Does `fetcher.py` or `index_engine.py` acquire the data robustly (handling rate limits and fallbacks)?
2. **Transform:** Is the data efficiently processed in memory using pandas or Polars before hitting the database?
3. **Save:** Is the data handed off exclusively to `librarian.py` for persistent storage in DuckDB?
4. **Use:** Does `screener.py` or `engine.py` query the stored data correctly rather than triggering redundant API calls?

## 3. Expanded Research Mandate
Do not write complex financial logic or scrapers from scratch without researching first. ALWAYS consult the following sources:
- **GitHub / Open Source:** Prioritize existing resilient logic (e.g., `nsepython`, `screener-scraper`, `OpenStockIndia`).
- **Official Broker APIs:** Verify execution and historical data endpoints strictly against official Upstox and Shoonya documentation.
- **Trading Forums (TradingQnA / Reddit):** Search Zerodha's TradingQnA, `r/algotrading`, or `r/IndianStreetBets` for translating visual scanner logic (like Chartink screens) into Python, or finding immediate workarounds for broken NSE endpoints.
- **Quantitative Integrity:** Validate mathematical formulas, factor rankings, and risk models (like ATR) via Quantitative Finance Stack Exchange.

## 4. Universe Tiering & Compaction
